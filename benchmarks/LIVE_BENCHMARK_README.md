# Live Agent Benchmark - Head-to-Head LLM Testing

## Overview

This benchmark is designed to test LLM agents in a live setting using real API documentation across 5 protocol types. Each agent receives identical tasks but with different documentation formats: verbose (raw spec) vs. LAP (compressed).

## Generated Assets

**Script**: `benchmarks/live_benchmark.py`  
**Output**: `benchmarks/live_prompts/` (20 prompt files)

## Coverage

### Protocol Distribution
- **OpenAPI**: 2 docs (Stripe Charges, GitHub Core)
- **GraphQL**: 2 docs (GitHub, Stripe)
- **AsyncAPI**: 2 docs (Stripe Webhooks, Slack Events)
- **Postman**: 2 docs (Stripe Payments, GitHub Repos)
- **Protobuf/gRPC**: 2 docs (User Service, Chat Service)

**Total**: 10 docs × 2 versions = 20 prompts

### Task Distribution
- Each doc includes **10 tasks** (100 tasks total)
- Tasks cover common API operations (CRUD, list, search, etc.)
- Tasks are protocol-specific (REST calls, GraphQL queries, gRPC RPCs, etc.)

## Prompt Structure

Each prompt follows this format:

```
You are an API integration assistant. You will be given API documentation and a series of tasks.
For each task, write the exact API call (curl command for REST, GraphQL query, protobuf RPC call, etc).

Format your response as:
TASK 1: <your api call>
TASK 2: <your api call>
... etc

--- API DOCUMENTATION ---
<doc here>
--- END DOCUMENTATION ---

TASKS:
1. <task 1>
2. <task 2>
...
10. <task 10>
```

## File Naming Convention

- `verbose_<protocol>_<index>.txt` - Raw spec file (original format)
- `lap_<protocol>_<index>.txt` - Compiled LAP version

Example:
- `verbose_openapi_0.txt` - Stripe Charges (raw OpenAPI spec)
- `lap_openapi_0.txt` - Stripe Charges (LAP compiled)

```bash
ls -lh benchmarks/live_prompts/
```
